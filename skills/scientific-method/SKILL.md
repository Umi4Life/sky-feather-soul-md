# Scientific Method Skill

## Purpose

Use this skill for investigations, debugging, unknown behavior, disputed hypotheses, and design questions where evidence matters more than confidence.

## Trigger Conditions

Use when:

- behavior is unexplained
- multiple causes are plausible
- logs or symptoms disagree with assumptions
- a fix needs verification
- a debate can be resolved by observation

## Method

1. **Observe.** Gather symptoms, logs, inputs, outputs, environment, and reproduction details.
2. **Question.** Define the exact unknown. Avoid solving the wrong problem.
3. **Hypothesize.** List plausible causes and rank them by evidence, not vibes.
4. **Experiment.** Choose the smallest test that can change your belief.
5. **Analyze.** Compare expected vs actual results.
6. **Iterate.** Update the model and repeat if needed.
7. **Record.** Capture findings if the result will matter later.

## Rules

- Arguments are not evidence.
- Authority is not evidence.
- Observable results carry the highest weight.
- A failed experiment is useful if it narrows the search space.
- Label unknowns honestly.

## Verification

Before concluding, ask:

- Did the experiment actually test the hypothesis?
- Is there a simpler explanation?
- Can the result be reproduced?
- What evidence would falsify this conclusion?
