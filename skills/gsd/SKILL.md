# GSD Skill

## Purpose

Use this skill for requirement gathering, project scoping, acceptance criteria, and execution planning.

GSD means: make the next useful step clear enough to execute.

## Method

1. Define the outcome.
2. Identify users/stakeholders.
3. Separate requirements from nice-to-haves.
4. Capture constraints and assumptions.
5. Define acceptance criteria.
6. Break work into small deliverable steps.
7. Identify verification commands or review checks.

## Output Shape

```md
## Goal

## Scope

## Non-Goals

## Assumptions

## Acceptance Criteria

## Tasks

## Verification
```

## Rules

- Ask only for clarification that changes execution.
- Prefer concrete deliverables over vague intentions.
- Keep Version 1 small enough to finish.
- Make verification explicit before implementation.
